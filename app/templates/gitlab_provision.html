GitLab Assessment Provisioning

Assessments are generated by selecting the course structure, source repository, assessment type, and student roster before running a non-destructive dry run.

How assessments will be generated

  1. 1
    Choose a course

    Select an existing GitLab course group or create a new course path.

  2. 2
    Choose an offering

    Select the delivery period, such as Autumn 2026, or create a new offering.

  3. 3
    Choose or create an assessment name

    This assessment name becomes the GitLab group where generated repositories will live.

  4. 4
    Choose a base repository

    Start repositories blank or fork them from an existing course project.

  5. 5
    Choose individual or group assessment

    Individual assessments create one repository per student. Group assessments use the CSV group name as the repository name.

  6. 6
    Upload the roster CSV and run the dry run

    The dry run validates the target, memberships, and repository actions before any provisioning is performed.
